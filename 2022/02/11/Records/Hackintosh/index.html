【折腾记录】黑苹果的种植记录

发表于2022-02-11|更新于2022-07-27|折腾记录
|字数总计:3.7k|阅读量:

Warning:

该文章不定期更新，可能会有部分内容已经过时，请谨慎参考。

选择折腾黑苹果的理由有很多，无论是使用强大的Final Cut Pro处理视频工作流，还是使用XCode开发IOS等苹果系应用，抑或者是为了享受丝滑舒服的动效和清爽统一的风格设计，甚至可能是为了能在同龄人的圈子中脱颖而出、招摇过市。但追根究底，选择另辟蹊径的最根本动力依旧是Mac设备那劝退的高昂价格。

无论如何，我们要感谢苹果与英特尔此前达成的合作——使用英特尔X86平台，这为在非苹果设备上安装使用MacOS提供了可能。不过遗憾的是，2020年苹果发布了搭载基于ARM架构的M1芯片的MacBook，至此二者合作宣告结束，也为黑苹果划定了死期。所以，黑苹果终将成为历史，且折腾且珍惜。

网络上教程很多，信息量也非常大，其中又不乏大量的英文文档，所以在此简单总结一下我的折腾经历。

种植声明 （劝退声明）

在折腾之前，首先要搞清楚以下几点。

  1. 黑苹果大概率会存在明显影响体验的问题，处于“能用”级别的黑苹果不如使用Windows来得舒服踏实。

  2. “能用”与“近乎完美”之间的距离就是“永无止境的折腾”，再“完美”的黑苹果也是“黑”苹果。

  3. 黑苹果的实现原理可以简单理解为通过仿冒苹果设备让系统认为它跑在正统的苹果机器上。所以，意图运行黑苹果的设备必须满足的基本条件是硬件信息尽可能接近已发布的苹果产品。

    可参考黑苹果星球文章：《SIMBIOS ID、Platform-ID机型速查表》

  4. 请事先调查硬件是否存在运行黑苹果的可能。在实际情况允许的情况下，可更换或增加部分设备以满足运行需求。

    可参考黑苹果星球的《2022年黑苹果硬件配置推荐表》，同时请留意远景论坛，热心玩家会第一时间将其运行成功的设备信息发布出来。

    在这里着重强调如下几点：

    1. 对于CPU：
    • Intel为最优解。
    • AMD虽然已有解决方案，但不够稳定易出问题，不推荐折腾。
    1. 对于GPU：

    • 驱动NVIDIA显卡只支持到第10代，且系统版本大多停留在10.13.6或者11.x，其后显卡型号与系统版本均无法驱动；

    • 弃用NVIDIA后，苹果转向使用AMD独显，故此存在部分AMD显卡可以免驱，有些型号可以使用仿冒显卡设备ID进行驱动；

    • 至于英特尔核显，UHD630及之前核显大多容易驱动，而使用新架构的第11代酷睿及之后CPU的核显需要一定操作才可驱动使用。

      • 对于显卡驱动支持等相关信息，可参考黑苹果星球文章：《2022年黑苹果macOS Big Sur/Monterey显卡支持列表，持续更新中》
      • 对于仿冒显卡ID以驱动显卡的教程，可参考黑苹果星球文章：《如何仿冒显卡设备ID以驱动显卡？》
    1. 对于硬盘：

    • 推荐使用固态硬盘（SSD），绝大多数可以安装黑苹果。

    • 不推荐使用机械硬盘（HDD）。数据读写速度慢，安装耗时过长（据说一天左右）。

      • 部分型号的SSD不建议安装黑苹果：（此处只列举博主知道的，其他型号可查阅上方链接）
      • 三星 PM961 / PM981 / PM981a / PM991：三星OEM产品，常见于品牌笔记本中。使用该SSD会导致MacOS无法稳定运行。可使用NVMeFix.kext（1.0.2版及以上） 修复，但情况依旧不乐观
      • 三星 970 EVO Plus：2019年5月前出厂的有可能存在问题，但可通过官方固件升级解决
    1. 对于WIFI&蓝牙：
    • 博通：由于苹果采用博通的无线解决方案，故博通卡相对容易驱动，部分免驱，甚至可以解锁隔空投送、接力等苹果服务。
    • 英特尔：2020年前无解，感谢@zxystd及其他开发者的努力与奉献，目前大部分芯片已基本解决无线上网与蓝牙基础功能使用问题，特定条件下支持接力，隔空投送尚不支持。（OpenIntelWireless项目地址）
    • 若意图更换网卡以解锁完整苹果协作服务，可参考黑苹果星球文章：《黑苹果无线网卡购买&安装&使用指南2022年版》。

  5. 选择已有成熟的、与自己设备相匹配的EFI能够大幅降低折腾成本。

  6. 由于电脑设备的特殊性，请谨慎对待任何一步深入设备底层的操作，杜绝“无知者无畏”。

  7. 黑苹果安装完成后，需留意引导、驱动等内容的更新日志。在准备更新系统版本前，请优先更新引导与驱动至对应版本。

  8. 善于使用搜索引擎，以及需要足够的时间精力和耐心来浏览帖子、解决问题。

种植说明（名词解释）

Q: 前文提到的“EFI”是什么？

全称即UEFI（Unified Extensible Firmware Interface，统一可扩展固件接口），为BIOS（Basic Input & Output System，基本输入输出系统）的升级替代方案，用于连接操作系统与底层硬件设备。目前绝大多数设备使用UEFI引导方案来启动操作系统，但在民间依然习惯使用“BIOS”来统称。

Q: 为什么黑苹果的核心在“EFI”上？

种植准备

文件准备

  1. 系统镜像文件（下载站点）：

    • 黑苹果星球

      需赞助超级会员才可下载使用，但超级会员可以解锁全站教程资源，性价比较高。

      该网站存在不同类型的镜像文件：

      • 官方镜像：即白苹果原版镜像，只能在苹果设备上使用，无法直接用于安装黑苹果。

      • 【推荐】带OpenCore、Clover、WEPE镜像：即U盘引导版镜像，可以直接制作U盘引导安装黑苹果。

        具体安装教程可参考黑苹果星球提供的《黑苹果macOS系统U盘版/恢复版基础安装教程》中的U盘部分。

      • 虚拟机CDR格式：适用于在Windows虚拟机（如VMware）上安装使用。

        在这里多说一句，由于苹果的政策，VMware在Windows上默认无法安装Mac系统。

        可以在网络中寻找破解安装工具，可参考黑苹果星球提供的解锁工具与MacOS虚拟机安装方法。

      • APFS恢复版镜像：适用于恢复镜像法安装黑苹果。

        具体安装教程可参考黑苹果星球提供的《黑苹果macOS系统U盘版/恢复版基础安装教程》中的恢复版部分。

      • 网络恢复镜像：前提是有线/无线网卡可驱动。

        具体安装教程可参考黑苹果星球提供的《黑果安装新姿势，使用网络恢复方法安装 macOS 的详细教程》。

    • 黑果小兵的部落阁

      需进入其微信公众号内，对相应文章进行打赏后才可获得下载链接推送。

    • 独行秀才的老窝

      博主所加 拯救者Y7000黑苹果群 的群主推荐的镜像来源，但博主从未使用过。

    • 远景论坛

      总有好心网友分享搬运镜像文件。

  2. 引导基础文件（GitHub链接）：

    • OpenCore
    • Clover

    二选一即可，一般来说推荐使用OpenCore（与Clover相比，OC有着众多优势）。

    虽然主流的黑苹果驱动已全面转向OC，而不再测试Clover的兼容性，但由于Clover自 r5122 版本开始集成OpenCore内核，驱动已可以正常更新使用。

    浅显来说，Clover与OpenCore肉眼可见的区别也就是文件目录、config文件配置方法以及开机引导界面不同。

    由于博主在安装时使用的为OpenCore引导，故以下内容均以OpenCore为主

  3. EFI文件：

    • 黑苹果星球：散装EFI文件 与 《各机型EFI文件合集》
    • 黑果小兵的部落阁：《Hackintosh黑苹果长期维护机型整理清单》
    • 远景论坛：散装EFI文件

    找到符合机型的合适的EFI是撬开黑苹果大门的第一步，请尽可能选择与机型最为匹配的、更新时间最近的EFI。

    最为匹配：主板、CPU、GPU（核显/独显）、WIFI&蓝牙、声卡型号一致（或笔记本型号相同）。

    如果以够用省心为标准，基本可以直接照搬该EFI作者提供的操作步骤，安装个系统就完事了。

    其余情况均需要亲自动手查阅资料与相关教程进行处理。

软件准备

  1. OpenCore引导下，config.plist编辑软件：

    • OpenCoreConfigurator

      第三方图形化编辑器，仅可用于Mac平台，使用时需要注意匹配OpenCore版本（留意更新日志），极不建议跨版本编辑。

      还可以使用其在Mac系统上挂载EFI分区，直接对EFI进行操作。

      编辑前注意进行文件备份，有小概率会出现破坏config文件情况。

    • OCAuxiliaryTools

      第三方图形化编辑器，支持跨平台（WIndows/Mac/Linux）编辑，使用时需要注意匹配OpenCore版本（留意更新日志），极不建议跨版本编辑。

      编辑前注意进行文件备份，有小概率会出现破坏config文件情况。

    • ProperTree

      基于 Python 的轻量级跨平台（WIndows/Mac/Linux）编辑器，推荐用于编辑 OpenCore 配置文件。

      虽然是图形化界面，不与底层配置代码打交道，但是是以“树”的形式展现配置信息。

      不会损坏config文件，但也建议提前备份。

  2. 【必备】黑苹果实用工具：Hackintool

    仅支持Mac平台，是使用黑苹果系统必不可少的实用工具软件，被誉为黑苹果的“瑞士军刀”。

  3. DSDT/SSDT 编译和反编译工具：Xiasl

    开源的跨平台 DSDT/SSDT 编译和反编译工具，可以反编译.aml文件生成.dsl文件，也可以对.dsl文件编译生成.aml文件。

    .aml文件，即 ACPI Machine Language File ，是ACPI（高级配置与电源接口，是一种操作系统电源管理、硬件配置接口的开放标准）二进制文件的格式名，在这里不妨理解为黑苹果的DSDT（存放在/EFI/OC/ACPI目录下）热补丁文件，使设备的控制尽量向白苹果靠近，解决稳定性并提升性能。

    .dsl文件，即.aml二进制文件的源文件，编译即可生成.aml文件。

    与 MaciASL 功能基本一致，但 Xiasl 可跨平台而 MaciASL 只能用于 Mac 系统下，并且 Xiasl 在各个方面的功能也更强大丰富些。

  4. 跨平台刻录工具：balenaEtcher

    跨平台，支持Windows、Linux、macOS平台。用于刻录 macOS 安装镜像（.dmg文件）。

  5. 硬盘管理软件：diskgenius

    一般使用它在Windows设备上对已经刻录好的U盘进行EFI的更换。

    还可以对磁盘进行分区、修复、备份以及其他高级操作。

  6. 跨平台USB定制工具：USBToolBox

    命令行工具，无GUI界面，可跨平台支持Windows，用于对USB端口进行定制。

    自 macOS Big Sur 11.3 更新后，OpenCore 的 USB 相关 Quirks XhciPortLimit 失效，导致黑苹果无法使用超出 15 个以上的 USB 接口。

    对于USB定制请参考黑苹果星球：《USB定制新姿势：Windows下定制黑苹果USB接口详细攻略》

注：若由于网络问题无法打开GitHub项目地址，可在黑苹果星球：黑苹果工具中搜索下载。

硬件准备

  1. 16GB U盘一个

    用途一：刻录镜像文件，以通过在BIOS设置U盘引导以安装黑苹果系统。

    用途二：在已装有黑苹果的电脑上测试EFI的可用性。

    使用 DiskGenius ，选中U盘，选择顶部菜单栏的 快速分区。

    在弹出的设置框内依次设置：

    • 创建分区个数：1个
    • 分区表类型：GUID
    • 创建新的ESP分区：否（去掉打钩）
    • 创建新的MSR分区：否（去掉打钩）
    • 分区格式：FAT32

    格式化完成后，将EFI拷贝进去。

    重启电脑，进入BIOS选择U盘启动，即可调用U盘中的EFI。

    用途三：定制USB。若在 Windows 下操作只需要准备一个 USB3.0 的U盘，若有 Type-C 接口再另准备相应接口的U盘即可；若在 Mac 下使用 Hackintool 操作，则需要 USB3.0 与 USB2.0 U盘各一个。

  2. 其他可以联网查资料以及允许使用U盘传输文件的设备

答疑解惑

  1. 远景论坛

    远景论坛是国内最大的黑苹果玩家聚集地，在这里有大佬发布各种驱动，也有跟你同样的玩家问问题、分享成果、总结经验。大部分情况下，你需要做的只是在搜索框输入问题关键词，在检索结果中寻找答案即可。

    查看、下载资源需要远景账号。

    账号注册需要邀请码，可在远景论坛的官方贴吧中寻找申请办法。

  2. 善用搜索引擎，基本可以屏蔽CSDN。

种植指南

机型信息

  • 型号：联想拯救者Y7000P 2019

  • 主板芯片组：Intel HM370

  • CPU：Intel Core i7-9750H 2.60GHz

  • GPU：

    • Intel UHD Graphics 630（核显驱动）

    • NVIDIA GeForce GTX 1660 Ti（独显无法驱动，已屏蔽）

  • SSD：

    • Samsung PM981A 1TB（OEM版不支持安装黑苹果，虽已有解决方案，但不建议使用，已屏蔽）
    • Crucial MX500 500G（博主后加的SATA固态，用来安装黑苹果）

  • WIFI：Intel Wireless-AC 9560 160MHz

  • 声卡：Realtek ALC257
